# Top 3 인수 조건

## 1. 예약 생성
인수 조건: "고객은 캠핑 사이트를 지정된 날짜에 예약하고, 중복되지 않으며 유효한 예약 정보를 시스템에 성공적으로 저장할 수 있어야 한다."

검증 포인트:
- [ ] 예약은 오늘로부터 30일 이내의 날짜에만 생성된다.
- [ ] 과거 날짜 또는 종료일이 시작일보다 빠른 경우 예약이 거부되어야 한다.
- [ ] 동일 사이트, 동일 기간에 대한 중복 예약이 방지되어야 한다.
- [ ] 예약자 이름, 전화번호 등 필수 정보가 누락되면 예약이 실패해야 한다.
- [ ] 예약 완료 시 6자리의 고유한 영숫자 확인 코드가 생성 및 반환되어야 한다.

## 2. 동시성 제어
인수 조건: "동일한 캠핑 사이트와 날짜에 대해 여러 예약 요청이 동시에 발생하더라도, 시스템은 단 하나의 예약만 성공시키고 나머지는 거부하여 중복 예약을 원천적으로 방지해야 한다."

검증 포인트:
- [ ] 동일 사이트, 동일 날짜에 대한 예약 요청을 동시에 여러 개 보냈을 때, 오직 1개의 요청만 성공하고 나머지는 실패해야 한다.
- [ ] 동시성 문제를 해결하기 위한 기술(예: 비관적/낙관적 락)이 적용되었고, 관련 멀티스레드 테스트 시나리오를 통과해야 한다.

## 3. 연박 예약
인수 조건: "고객은 시작일과 종료일을 지정하여 연박 예약을 요청할 수 있으며, 시스템은 해당 기간 내 모든 날짜가 예약 가능한 경우에만 예약을 성공시켜야 한다."

검증 포인트:
- [ ] 예약하려는 전체 기간 중 단 하루라도 이미 예약된 날짜가 포함되어 있으면 예약 전체가 실패한다.
- [ ] 연박 예약이 완료되면, 예약된 모든 날짜가 다른 사용자의 가용성 확인 시 '예약 불가'로 표시되어야 한다.
- [ ] 최대 예약 가능 기간(예: 30일)을 초과하는 연박 예약은 거부되어야 한다.
- [ ] 연박 예약 취소 시, 관련된 모든 날짜의 예약 상태가 정상적으로 취소되어야 한다.
